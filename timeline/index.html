The leap

Current × Prototype

Today the rep starts in PO upload, runs the entire flex flow in BEES One via spreadsheet, and only returns to Link at the end to add to the cart. The prototype keeps everything inside the Link cart, end to end.

⛔ Current product

PO upload → spreadsheet → whole flow in BEES One → back to Link → cart

  1. Start the order-taking flow via PO upload
  2. Download the spreadsheet
  3. Adjust price/discount in the spreadsheet
  4. Start the flow in BEES One
  5. Do everything and go through approval — in BEES One
  6. Return to Link
  7. Add to the cart in Link
Pain: the whole flex flow lives in BEES One — the rep only comes back to Link at the very end to add to the cart. Spreadsheet in the middle, context switch across 2 products, manual data reuse. Friction kills flex-pricing adoption.
✦ Prototype

Everything inside the Link cart, end to end

  1. Build the cart in BEES Link
  2. Save as draft (name the draft)
  3. Select the payment method
  4. Edit flex % per product / per store
  5. Save changes → summary (discounts / increases / final price)
  6. Send for approval → approval
  7. Place order — never leaving Link
Gain: group quotation is born in the cart (not the spreadsheet), Flex Wallet balance visible, data reuse via endpoint, less friction → higher adoption and shorter time-to-approved-quote.
